Overview

Campus Scheduler is a web-based platform designed to help businesses, particularly Small to Medium Enterprises (SMEs), Startups, and Tech companies, manage their workflows effectively through a role-based interface (Admin, Manager, Worker). Hosted on Railway, it facilitates project management, event scheduling, task assignment, and team communication.

Users can register directly or sign in using Google Authentication. The system leverages a PostgreSQL database for robust data storage and utilizes a tech stack chosen for rapid development and flexibility: Frontend: HTML, CSS, JavaScript. Backend: Python (Flask) and potentially Java, using REST APIs. Database: PostgreSQL.

The platform aims to provide a cost-effective solution, initially free, with potential for a future subscription model. It emphasizes rapid, iterative development using the RAD model to quickly adapt to client needs.

Key Features & Functionality

The platform offers a range of features tailored to improve organizational efficiency:

  • User Management: Secure registration, login (including Google Sign-In), and role assignment (Admin, Manager, Worker).
  • Project & Task Management: Managers can create projects, define tasks, assign them to workers, set deadlines, and track progress. Workers can view, update, and complete assigned tasks.
  • Scheduling & Calendars: Personal task calendars for employees and team task calendars for managers. Event creation and scheduling capabilities.
  • Communication: Real-time, role-aware messaging between colleagues and team leaders.
  • Dashboards & Visualization: Role-based dashboards displaying relevant information (projects, tasks, team info). A project dependency graph visualizes task connections.
  • Data Privacy: GDPR compliant data handling with options for users to request data deletion.

Challenges & Technical Considerations

Building Campus Scheduler involves addressing several technical and non-functional requirements to ensure a robust, secure, and user-friendly platform.

Scalability and Performance:
  • Challenge: Ensuring the system handles concurrent users (target: 100+) without performance degradation, especially for features like the real-time graph rendering (target: < 1s) and achieving high Google PageSpeed scores (target: 90+).
  • Solution: Utilizing efficient backend frameworks (Flask), optimizing PostgreSQL queries, implementing caching where appropriate, asynchronous task handling for heavy operations, and frontend performance optimizations. Load testing will be crucial.
Security and Data Integrity:
  • Challenge: Protecting user data, ensuring secure authentication, preventing common web vulnerabilities, and guaranteeing data availability.
  • Solution: Implementing HTTPS (SSL/TLS), OAuth 2.0 for Google Sign-In, secure password hashing, input validation to prevent SQL injection/XSS, regular database backups (NFR5), and adhering to GDPR principles for data handling.
User Experience (UX) and Responsiveness:
  • Challenge: Creating an intuitive interface that works seamlessly across desktop and mobile devices (NFR3), managing complexity arising from different user roles and features.
  • Solution: Employing responsive web design principles (HTML/CSS/JS), designing clear role-based dashboards and navigation, usability testing, and leveraging Flask templates for dynamic content rendering.
Rapid Development (RAD) Management:
  • Challenge: Balancing the speed of the RAD model with the need for code quality, maintainability, and thorough testing, especially with a potentially distributed team.
  • Solution: Establishing clear coding standards, using version control (e.g., Git), implementing unit and integration tests, regular code reviews, and maintaining clear communication channels within the team.
